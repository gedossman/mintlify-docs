## Quick Start

Tell your coding agent:

```
use npx interf to create a readiness spec for Nike deployment
```

That's it. Your coding agent installs the toolkit, scans your codebase, extracts an `interf.yaml` manifest declaring every dependency, and simulates enterprise readiness — all in one prompt.

Works with **Claude Code**, **Codex**, **Cursor**, and **Goose**.

<Note>
  **No API keys. No network calls. No code execution.** Interf installs skills — structured domain knowledge — to your coding agent. The intelligence comes from your agent; Interf teaches it the Agent Readiness Protocol.
</Note>

## Who It's For

Interf provides a toolkit for **vendor deploy engineers and teams responsible for enterprise deployments**. If you're shipping an AI agent into Fortune 500 environments, Interf helps you:

- **Extract** every enterprise dependency from your codebase — APIs, databases, auth, secrets, compliance, approvals
- **Review** the draft manifest before it becomes your deployment contract
- **Simulate** readiness against specific enterprise profiles — Goldman Sachs, Nike, Series B startup — and get timelines, stakeholders, risks, and blockers
- **Check enterprise readiness** for your solution within 5 minutes

## What You Get

Each simulation produces per-dependency analysis:

- **Resolution flow** — ordered steps with owners, timelines, and blockers
- **Stakeholders** — roles, involvement level, time commitment
- **Risks** — likelihood, impact, and mitigations
- **Critical path** — total days to production readiness

```yaml
# interf.yaml — your agent's deployment contract
name: crm-automation-agent
version: 0.1.0
description: Automates CRM data entry and follow-up scheduling

dependencies:
  - type: api
    name: Salesforce API
    description: Read/write contacts and opportunities
    required: true
    config:
      endpoint: https://api.salesforce.com
      auth_type: oauth2
      scopes: [api, refresh_token]

  - type: auth
    name: SSO Integration
    description: Enterprise SSO for agent authentication
    required: true
    config:
      provider: okta
      protocol: saml

  - type: compliance
    name: SOC2 Compliance
    description: Required for handling customer PII
    required: true
    config:
      frameworks: [SOC2]
      controls: [data_encryption, audit_logging, access_review]

  - type: approval
    name: Security Review
    description: InfoSec team must approve API access
    required: true
    config:
      approver_role: security_engineer
      justification: Agent accesses customer PII via Salesforce

capabilities:
  - read_contacts
  - write_contacts
  - read_opportunities
  - schedule_followups

risk_level: high
```

## The Protocol

Today, agents deploy blindly into enterprise environments. Readiness gaps — missing data access, authentication, compliance permissions — are discovered mid-flight, forcing vendors and enterprises to fix them in production. Forward-deployed engineers spend months on discovery and dependency resolution that should have happened before deployment began.

**Interf provides the structural fix: Readiness-as-Code.** The Agent Readiness Protocol is an open standard for declaring what agents need from enterprises before deployment. Vendors define it. Enterprises resolve it. Agents verify it.

### How It Works in Production

The open-source toolkit is day zero — vendors adopt the protocol, define their spec, and publish to the Interf registry. Enterprises pull the manifest and self-prepare for agent onboarding. The production flow eliminates the 6+ months of discovery, dependency resolution, and back-and-forth that stalls enterprise deployments today.

1. **Vendor** defines `interf.yaml` and publishes to the Interf registry — the deployment contract declaring every dependency.
2. **Enterprise** loads the vendor manifest into the Interf Gateway, creating a readiness workspace.
3. **Gateway** exposes readiness state as an MCP endpoint — each dependency tracked from unresolved to verified.
4. **Enterprise IT** resolves dependencies — provisions API access, configures SSO, completes security reviews — with each resolution audited.
5. **Vendor agent** connects to the enterprise gateway endpoint, queries its readiness state, and self-verifies each dependency against vendor criteria.
6. **Agent confirms** all dependencies verified — ready for production. Full audit trail of resolution for every item.

The gateway is the standardized interface between vendor and enterprise. Resolution happens on the enterprise side. Verification happens on the agent side. Interf connects the two — with an independent, auditable handshake that establishes who is responsible for what.

Vendors who publish to the registry can rely on enterprises implementing the protocol. Enterprises who adopt the gateway can onboard any vendor agent that speaks Interf. The result: agent self-onboarding without human intervention, within the eligible scope defined by the contract.
